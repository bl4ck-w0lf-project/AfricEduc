# AfricEduc — Logique de gestion des écoles, classes et programmes pédagogiques

# 1. Objectif général

AfricEduc est une plateforme SaaS de gestion scolaire multi-écoles.

Lorsqu'une école est créée dans le système, AfricEduc doit automatiquement initialiser une structure scolaire de base afin d'aider l'administrateur dans sa configuration.

Cette initialisation comprend :

* les niveaux scolaires ;
* les séries ;
* les programmes pédagogiques (`curricula`) ;
* les matières avec leurs coefficients ;
* les classes par défaut.

L'objectif est d'éviter que l'administrateur parte d'une configuration vide.

Après la création d'une école, l'administrateur doit avoir une base prête à personnaliser :

```
École créée
      |
      |
      +── Niveaux scolaires
      |
      +── Séries
      |
      +── Curricula
      |
      +── Matières + coefficients
      |
      +── Classes par défaut
```

---

# 2. Gestion des cycles scolaires

AfricEduc distingue deux cycles :

* Premier cycle (collège)
* Second cycle (lycée)

Chaque cycle possède une logique différente dans la gestion des classes.

---

# 3. Premier cycle

Le premier cycle correspond aux niveaux :

* 6ème
* 5ème
* 4ème
* 3ème

Dans ce cycle :

* `level_id` est utilisé pour identifier le niveau.
* `serie_id` reste NULL.
* `group_name` représente le nom de la classe.

Exemple :

```
6ème A
6ème B
6ème C
6ème D
```

Stockage :

```
level_id = 6ème
serie_id = NULL
group_name = A
```

Signification :

```
level_id → Niveau scolaire
group_name → Nom de la classe
```

---

## Création automatique des classes premier cycle

Lors de la création d'une école, AfricEduc crée automatiquement :

### 6ème

```
6ème A
6ème B
6ème C
6ème D
```

### 5ème

```
5ème A
5ème B
5ème C
5ème D
```

### 4ème

```
4ème A
4ème B
4ème C
4ème D
```

### 3ème

```
3ème A
3ème B
3ème C
3ème D
```

Total :

```
4 niveaux × 4 classes = 16 classes
```

---

# 4. Second cycle

Le second cycle correspond aux niveaux :

* Seconde
* Première
* Terminale

Dans ce cycle :

* `level_id` n'est pas utilisé.
* `serie_id` représente la série.
* `group_name` représente le niveau scolaire.

Les séries disponibles sont :

```
A
B
C
D
```

---

Exemple :

Classe :

```
Terminale D
```

Stockage :

```
level_id = NULL
serie_id = D
group_name = Terminale
```

Signification :

```
serie_id → Série
group_name → Niveau scolaire
```

---

## Création automatique des classes second cycle

AfricEduc crée :

### Seconde

```
Seconde A
Seconde B
Seconde C
Seconde D
```

### Première

```
Première A
Première B
Première C
Première D
```

### Terminale

```
Terminale A
Terminale B
Terminale C
Terminale D
```

Total :

```
3 niveaux × 4 séries = 12 classes
```

---

# 5. Total des classes créées automatiquement

Lorsqu'une école est créée :

Premier cycle :

```
16 classes
```

Second cycle :

```
12 classes
```

Total :

```
28 classes par défaut
```

Ces classes peuvent ensuite être modifiées ou supprimées par l'administrateur.

---

# 6. Structure de la table classes

Table :

```sql
classes
```

Structure :

```
id
school_id
level_id
serie_id
group_name
max_students
academic_year
created_at
```

---

## Description des champs

### school_id

Identifie l'école propriétaire de la classe.

---

### level_id

Utilisé uniquement pour le premier cycle.

Exemple :

```
6ème
5ème
4ème
3ème
```

---

### serie_id

Utilisé uniquement pour le second cycle.

Valeurs :

```
A
B
C
D
```

---

### group_name

Représente le nom de la classe.

Premier cycle :

```
A
B
C
D
```

Exemple :

```
6ème + A = 6ème A
```

Second cycle :

```
Seconde
Première
Terminale
```

Exemple :

```
Terminale + D = Terminale D
```

---

### max_students

Représente la capacité maximale de la classe.

Exemple :

```
6ème A
max_students = 50
```

Le nombre réel d'élèves n'est pas stocké.

Il est calculé depuis les inscriptions.

---

# 7. Gestion des programmes pédagogiques (Curricula)

Le curriculum représente le programme scolaire.

Il définit :

* les matières enseignées ;
* les coefficients ;
* les règles utilisées pour les calculs de notes.

Une classe ne possède pas directement ses matières.

Elle récupère automatiquement son curriculum.

---

# 8. Table curricula

Structure :

```
curricula

id
school_id
level_id
serie_id
name
created_at
updated_at
```

---

# Curriculum premier cycle

Exemple :

Programme 6ème :

```
school_id = 19
level_id = 6ème
serie_id = NULL
name = Programme 6ème
```

Programme 3ème :

```
school_id = 19
level_id = 3ème
serie_id = NULL
name = Programme 3ème
```

---

# Curriculum second cycle

Le programme dépend de la série.

Exemple :

Programme Terminale D :

```
school_id = 19
level_id = NULL
serie_id = D
name = Programme Terminale D
```

---

# 14. Matières de la classe

Très important.

Dans le détail de la classe, AfricEduc doit afficher les matières enseignées.

Exemple :

Classe :

```

```

Le système recherche automatiquement :

```
curricula
WHERE serie_id = D
```

```
curriculum_subjects
```

# 15. Relations finales

Architecture :

```
SCHOOLS
   |
   |
   +----------------+
   |                |
   |                |
CLASSES          CURRICULA
   |                |
   |                |
STUDENTS     CURRICULUM_SUBJECTS
                    |
                    |
                SUBJECTS
```

---

# 16. Fonctionnement global

Création d'une école :

```
Création école
       |
       |
       +--> Création niveaux
       |
       +--> Création séries
       |
       +--> Création curricula
       |
       +--> Association matières + coefficients
       |
       +--> Création classes par défaut
```

Ensuite :

```
Classe créée
      |
      |
      +--> Recherche curriculum correspondant
      |
      +--> Récupération des matières
      |
      +--> Application des coefficients
      |
      +--> Gestion des notes et bulletins
```

---

# Résumé métier final

Dans AfricEduc :

* `classes` représente les groupes d'élèves.
* `curricula` représente les programmes pédagogiques.
* `curriculum_subjects` représente les matières et coefficients.
* Une classe récupère automatiquement son programme.
* L'administrateur n'a pas besoin de configurer les matières classe par classe.
* La création d'une école génère automatiquement une structure scolaire complète.

Cette conception permet à AfricEduc d'être un SaaS multi-écoles flexible, configurable et adapté au système scolaire béninois.
